# Book English Tutor (отдельный проект в ветке `BOOK`)

Локальный сайт для загрузки PDF-книги и генерации:
- плана уроков по **конкретной странице**;
- набора уроков в Markdown по этой же странице;
- сохранения всех результатов на вашем ПК между сессиями.

> В проекте используется Google Gemini API (модель `gemini-3.1-flash-lite`) через переменную окружения `GOOGLE_API_KEY`.
> Для «thinking level high» включён `thinking_budget` (по умолчанию `24576`, настраивается через `GEMINI_THINKING_BUDGET`).

---

## Что реализовано по вашему ТЗ

- Загрузка PDF в веб-интерфейсе.
- Слева вертикальная лента страниц (миниатюры + прокрутка).
- Клик по странице выбирает её для работы.
- Кнопка **«Сгенерировать план»**:
  - отправляет **одну выбранную страницу** в модель;
  - в prompt вставляется **буквально ваш текст без изменений**;
  - сохраняет `plan.md` для выбранной страницы.
- Кнопка **«Сгенерировать уроки»**:
  - использует `plan.md`;
  - генерирует уроки (`lesson_01.md`, `lesson_02.md`, ...);
  - для каждого следующего урока передаёт контекст из последних уроков (упрощённая память внутри серии по странице).
- В интерфейсе рядом с выбранной страницей видны:
  - ссылка на план;
  - раскрывающийся список уроков.
- Все данные сохраняются в `book/data/books/...` и доступны после перезапуска приложения.

---

## Важное про лимиты API (500/день, 15/мин)

В текущей версии запросы отправляются напрямую, поэтому:
- не запускайте генерацию для большого числа страниц одновременно;
- делайте паузы между страницами и уроками;
- при необходимости можно добавить троттлинг/очередь запросов (следующий этап).

## Архитектура (кратко)

- `app.py` — FastAPI backend + генерация страниц/планов/уроков.
- `templates/index.html` — UI.
- `static/app.js` — фронтенд-логика.
- `static/styles.css` — стили.
- `data/books/<book_id>/`:
  - `source.pdf`
  - `pages/page_XXXX.png` — изображения страниц
  - `page_XXXX.txt` — извлечённый текст
  - `page_XXXX/plan.md`
  - `page_XXXX/lesson_YY.md`

## Установка и запуск

## 1) Требования

- Python 3.11+
- Linux/macOS/WSL (на Windows тоже можно, но команды ниже в unix-стиле)

## 2) Установка зависимостей

```bash
cd book
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 3) Настройка API ключа Google

```bash
export GOOGLE_API_KEY="ВАШ_КЛЮЧ"
export GEMINI_THINKING_BUDGET="24576"
```

Проверить, что ключ установлен:

```bash
echo $GOOGLE_API_KEY
```

Проверка runtime-конфига (модель + thinking budget):

```bash
curl http://localhost:8000/api/runtime-config
```

## 4) Запуск приложения

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

Открыть в браузере:

- `http://localhost:8000`

---

## Как пользоваться

1. Нажмите **Загрузить PDF** и выберите книгу.
2. В левой панели кликните по нужной странице.
3. Нажмите **Сгенерировать план**.
4. После этого нажмите **Сгенерировать уроки**.
5. Открывайте `plan.md` и уроки по ссылкам в интерфейсе.

---

## Где хранятся результаты

Всё хранится локально в:

```text
book/data/books/
```

Удаление книги = удаление соответствующей папки `book_id`.

## Как скачать проект с GitHub
